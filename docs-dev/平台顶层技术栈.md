# [PolarisNexus](https://github.com/PolarisNexus)平台顶层技术栈

> 地基文档 -- 面向 AI 开发参考，不解释原理，原理另建专项文档。
>
> 旨在构建一个**基于多语言微服务（Polyglot Microservices）的企业级技术底座**。

---

## 1 架构概述

多语言微服务 + 可组合架构。用成熟开源组件拼装平台基座，业务以独立产品接入。

**一个平台基座 + 多产品矩阵**

- **平台基座:** 统一接管 IAM、网关、多租户路由、可观测性、容器运行环境。
- **产品接入:** Java 业务群、Python AI 群及其他语言模块，通过 OpenAPI 规范、HTTP/gRPC 协议对接，独立部署。

### 架构分层

| 层 | 职责 | 核心组件 |
|---|---|---|
| 应用产品层 | 业务功能 | Spring Boot 服务、Python AI 服务、前端应用 |
| 公共基座层 | WAF + 网关 + IAM | SafeLine、APISIX、Casdoor |
| 基础设施层 | 存储 / 缓存 / 搜索 | PostgreSQL、MinIO、Redis、Elasticsearch |

## 4 统一身份认证与网关

### 4.1 IAM -- Casdoor

- 全局用户目录、SSO 单点登录、JWT 令牌颁发
- 各子系统仅需信任并解析 JWT

### 4.2 API 网关 -- APISIX

系统唯一外部入口，接管所有 HTTP/gRPC 流量。

职责: JWT 校验、限流熔断、跨域处理、日志埋点。

双模式校验:
- 离线公钥校验 -- 面向高并发
- 实时穿透校验 -- 面向高敏感

**JWT 透传流程:**

```
             +-----------------+
             | Client (Web/App)|
             +--------+--------+
                      |
                      | 1. HTTPS 请求
                      v
             +--------+--------+
             |  SafeLine WAF   |
             | (恶意流量拦截)    |
             +--------+--------+
                      |
                      | 2. Bearer <JWT>
                      v
+-----------------------------------------------+
|               APISIX Gateway                  |
|                                               |
|  +----------------+    3. 获取公钥/校验签名     |     +---------------+
|  | JWT 鉴权 Plugin +---------------------------+---->|    Casdoor    |
|  +-------+--------+    4. 校验并返回结果        |<----|  (IAM 中心)   |
|          |                                    |     +---------------+
|          | 5. 提取 Token Payload              |
|          v                                    |
|  +----------------+                           |
|  | Header 转换重写  |                           |
|  | (X-User-ID etc)|                           |
|  +-------+--------+                           |
+-----------+-----------------------------------+
            |
            | 6. 转发 (携带身份 Header)
            v
     +------+-------+---------------+
     v              v               v
+---------+   +---------+    +---------+
| Spring  |   | Python  |    |  其他   |
| 业务服务 |   | AI 服务  |    | 扩展服务 |
+---------+   +---------+    +---------+
```

### 4.3 流量防护 -- SafeLine 

部署在 APISIX 前的反向代理 WAF，所有外部流量先经 SafeLine 过滤再转发至 APISIX。

防护能力:

- OWASP Top 10 (SQL 注入、XSS、CSRF、路径遍历等)
- CC 攻击防护 / 速率限制
- 爬虫识别与人机验证
- IP 黑白名单
- 个人版可支撑 800 QPS 的请求量

部署方式: Docker Compose，与平台其他组件统一编排。

职责分离: SafeLine 负责恶意流量拦截，APISIX 负责业务路由与鉴权。

### 4.4 内部服务间通信

- 协议: gRPC (HTTP/2 + Protobuf)
- 同一内网微服务互信，不重复鉴权

### 4.5 网关路由配置

- 初期: 声明式 YAML (DB-less / Standalone Mode)
- 未来: etcd 动态路由 (见路线图)

### 4.6 服务发现

- 初期: Docker Compose 直连 / K8s CoreDNS
- 不引入额外服务发现组件

## 5 数据存储与中间件

| 组件 | 用途 |
|---|---|
| PostgreSQL | 核心业务数据 |
| Elasticsearch | 全文检索、日志聚合、初期向量检索 |
| Redis | 热点缓存、分布式锁、简单异步任务 |
| MinIO | 统一对象存储 (S3 协议) |

### 5.1 向量数据库

设计上通过抽象层兼容多引擎 (Milvus、Qdrant、Elasticsearch)。

初期实现: Elasticsearch (dense_vector)。

### 5.2 异步任务

初期不引入消息队列。通过 Redis 或数据库轮询满足简单异步需求。

## 6 多租户数据隔离

- Casdoor 多租户 Realm，含 Client 配置和 RBAC 角色
- 网关层注入 `X-Tenant-ID`，下游服务据此隔离数据
- 方案: 共享数据库 + 租户字段过滤

---

## 7 安全架构

- WAF: SafeLine 前置拦截恶意流量 (OWASP Top 10 / CC / 爬虫)
- 认证: Casdoor 统一颁发 JWT，网关层校验
- 授权: RBAC，Casdoor 管理角色与权限
- 传输: HTTPS (外部)，内网 gRPC 互信
- 敏感数据: 环境变量 / Secret 管理，不硬编码

---

## 8 可观测性

Traces + Metrics + Logs 三支柱。

- 采集: OpenTelemetry SDK
- 后端: Elastic APM Server 接收 OTel 数据
- 存储与可视化: Elasticsearch + Kibana

---

## 9 高并发治理

- 限流: APISIX `limit-req` 插件
- 熔断: APISIX `api-breaker` 插件
- 负载均衡: K8s Service / Docker Compose 内置
- 自动扩容: K8s HPA

---

## 10 容器化部署

- **开发期:** Docker + Docker Compose 快速拉起全部组件
- **生产环境:** 离线 Kubernetes 集群编排
- **依赖锁定:** 离线环境严格锁版本
  - 前端: nvm + pnpm
  - Python: uv
  - 容器镜像: 构建时固化所有依赖
